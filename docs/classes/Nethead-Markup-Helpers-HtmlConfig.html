Class HtmlConfig.

Stores the configuration for the HTML Foundation. Everything can be overwritten at runtime as it is all public and static. See below for more documentation on configuration options.

$globalAttributes

HtmlConfig.php : 49

Global attributes supported by every HTML tag.

public static array<string|int, mixed> $globalAttributes = ['accesskey', 'class', 'contenteditable', 'dir', 'draggable', 'dropzone', 'hidden', 'id', 'lang', 'spellcheck', 'style', 'tabindex', 'title', 'translate']

If you have a need to quickly add some attribute on all tags, you can add it to global attributes list like this: HtmlConfig::$globalAttributes[] = 'yourTagName';. Remember: Not all attributes can be added on some elements according to HTML standard. It's your responsibility if the HTML will validate.

$voidElements

HtmlConfig.php : 23

HTML 5.3 void elements list.

public static array<string|int, mixed> $voidElements = ['area', 'base', 'br', 'col', 'embed', 'hr', 'img', 'input', 'link', 'meta', 'param', 'source', 'track', 'wbr']

Void elements (tags) are elements that can't have children elements and are closed right away. They still can have attributes. HTML Foundation is using this array to decide how and when the tag should be closed, and if it can have child elements. You can add more tags if you need to by calling HtmlConfig::$voidElements[] = 'yourTagName';
